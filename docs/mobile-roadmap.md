# Proofbase Mobile Roadmap

PASS 21 makes Proofbase installable and more mobile-ready through a PWA foundation. Native mobile apps should wait until the research workflow and saved-session model are proven.

## Mobile PWA Foundation

- Installable on Android through Chrome and compatible browsers.
- Touch-friendly search modes, evidence cards, and workspace navigation.
- App-like metadata, theme colors, icons, and offline-safe shell.
- Mobile layouts should prioritize search, saved research, and quick source checks.

## Mobile Product Direction

- Fast claim and URL checks from share sheets.
- Saved research sessions for later reading.
- ProofMedia feed with evidence clearly separated from opinion.
- Debate participation with compact evidence attachment tools.
- Routine updates and push notifications after permissions and backend strategy are stable.

## Mobile UX Priorities

- Large search target on first screen.
- Single-column result flow with collapsible source context.
- Bottom-friendly controls for search modes and filters.
- Compact evidence cards with source quality visible before discussion.
- Composer flows that discourage unsupported claims and require source context.

## Native App Later

- Evaluate React Native only after PWA usage proves mobile demand.
- Keep authentication, saved sessions, and evidence schemas shared with the web app.
- Add push notifications only for opted-in routines, followed topics, and saved debates.
